# Discount Service

The discount service provides the bounded context `Discount`.
It is responsible for managing discounts, coupons, and their usages by users.

## Domain Model

<CustomImage path="/renderedDiagrams/discountDomainModel" width="1542.5" height="663.25" />

:::info

Imported entities from other bounded contexts are marked with a dashed border.

:::

## Technology Stack

- **Language**: Kotlin
- **Framework**: Spring Boot
- **GraphQL Library**: GraphQL Kotlin
- **Database**: PostgreSQL
- **Schema Management**: Flyway

## Events

This section lists events that are defined or used by the discount service.

### Published Events

<details>
    <summary>
#### discount/discount/created
    </summary>

This event is published when a new discount is created.

```tsx
{
  id: string,
  discount: number,
  maxUsagesperUser?: number,
  validUntil: string,
  validFrom: string,
  minOrderAmount?: number,
  discountAppliesToCategoryIds: string[],
  discountAppliesToProductIds: string[],
  discountAppliesToProductVariantIds: string[]
}
```
</details>

<details>
    <summary>
#### discount/discount/updated
    </summary>

This event is published when a discount is updated.

```tsx
{
  id: string,
  discount: number,
  maxUsagesperUser?: number,
  validUntil: string,
  validFrom: string,
  minOrderAmount?: number,
  discountAppliesToCategoryIds: string[],

  discountAppliesToProductVariantIds: string[]
}
```
</details>

<details>

<details>

#### discount/order/validation-succeeded
</summary>

This event is thrown when the validation of an order caused by inventory/product-item/reservation-succeeded (TODO link) is successful.

</details>

<details>
<summary>

</details>


### Subscribed Events

- [inventory/product-item/reservation-succeeded](/docs/docs/services/inventory#inventoryproduct-itemreservation-succeeded): Causes a validation of the order to check if the discounts can still be applied.
- [user/user/created](/docs/docs/services/user#userusercreated): Used to import `User` from the `User` bounded context.
- [catalog/category/created](/docs/docs/services/catalog#catalogcategorycreated): Used to import `Category` from the `Category` bounded context.
- [catalog/product/created](/docs/docs/services/catalog#catalogproductcreated): Used to import `Product` from the `Catalog` bounded context.
- [catalog/product-variant/created](/docs/docs/services/catalog#catalogproduct-variantcreated): Used to import `ProductVariant` from the `Catalog` bounded context.
