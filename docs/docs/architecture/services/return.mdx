# Return Service

The return service provides the bounded context `Return`.
It is responsible for managing and validating returns.

## Domain Model

<CustomImage path="/renderedDiagrams/returnDomainModel" width="883.9301563617175" height="680.25" />

:::info

Imported entities from other bounded contexts are marked with a dashed border.

## Technology Stack

- **Language**: Kotlin
- **Framework**: Spring Boot
- **GraphQL Library**: GraphQL Kotlin
- **Database**: PostgreSQL
- **Schema Management**: Flyway

## Events

This section lists events that are defined or used by the return service.

### Published Events

<details>
<summary>
#### return/return/created
</summary>

This event is published when a new user return is created.

```tsx
{
  id: string,
  orderId: string,
  orderItemsIds: string[],
  reason: string,
  refundedAmount: number,
  createdAt: string
}
```
</details>

### Subscribed Events

- [shipment/shipment/created](/docs/docs/architecture/services/shipment#shipmentshipmentcreated): Used to import `Shipment` from the `Shipment` bounded context.
- [shipment/shipment/status-updated](/docs/docs/architecture/services/shipment#shipmentshipmentstatus-updated): Used to update the delivery status of imported `Shipment` entities.
- [order/order/created](/docs/docs/architecture/services/order#orderordercreated): Used to import `Order` and associated `OrderItem`s from the `Order` bounded context.
- [catalog/product-variant-version/created](/docs/docs/architecture/services/catalog#catalogproduct-variant-versioncreated): Used to import `ProductVariantVersion` from the `Product` bounded context.
