# Shipment Service

The shipment service provides the bounded context `Shipment`.
It is responsible for managing shipments and their state, and interacts with an external shipment provider to do so.

## REST API

The service provides a REST API which is used by the external shipment provider to inform about shipment status changes.

### POST `/shipment/{id}/status`

Updates the status of a shipment.

**Body**

```tsx
{
  status: ShipmentStatus;
}

enum ShipmentStatus {
  PENDING,
  IN_PROGRESS,
  DELIVERED,
  FAILED
}
```

## Technology Stack

- **Language**: Kotlin
- **Framework**: Spring Boot
- **GraphQL Library**: GraphQL Kotlin
- **Database**: PostgreSQL
- **Schema Management**: Flyway

## Events

This section lists events that are defined or used by the shipment service.

### Published Events

<details>
<summary>
#### shipment/shipment/created
</summary>

This event is published when a new shipment is created.

```tsx
{
  id: string,
  orderId?: string,
  returnId?: string,
  status: ShipmentStatus,
  orderItemIds: string[],
  shipmentMethodId: string,
  shipmentAddressId: string
}
```
</details>

<details>
<summary>
#### shipment/shipment/creation-failed
</summary>

This event is published when the creation of a shipment failed.

```tsx
{
  orderId?: string,
  returnId?: string,
  orderItemIds: string[],
  shipmentMethodId: string,
  shipmentAddressId: string,
  reason: string
}
```
</details>

<details>
<summary>
#### shipment/shipment/status-updated
</summary>

This event is published when the status of a shipment is updated.

```tsx
{
  id: string,
  orderId?: string,
  returnId?: string,
  status: ShipmentStatus,
  orderItemIds: string[],
}
```
</details>

<details>
<summary>
#### shipment/shipment-method/created
</summary>

This event is published when a new shipment method is created.

```tsx
{
  id: string,
  name: string,
  description: string,
  ref: string,
  baseFees: number,
  feesPerItem: number,
  feesPerKg: number
}
```
</details>

<details>
<summary>
#### shipment/shipment-method/archived
</summary>

This event is published when a shipment method is archived.

```tsx
{
  id: string;
}
```
</details>

### Subscribed Events

- [payment/payment/payment-enabled](/docs/docs/architecture/services/payment#paymentpaymentpayment-enabled): Used to create new shipments for an order when the order and associated payment is validated successfully.
- [address/user-address/created](/docs/docs/architecture/services/address#addressuser-addresscreated): Used to import `UserAddress` from the `Address` bounded context.
- [address/vendor-address/created](/docs/docs/architecture/services/address#addressvendor-addresscreated): Used to import `VendorAddress` from the `Address` bounded context.
- [catalog/product-variant-version/created](/docs/docs/architecture/services/catalog#catalogproduct-variant-versioncreated): Used to import `ProductVariantVersion` from the `Catalog` bounded context.
- [return/return/created](/docs/docs/architecture/services/return#returnreturncreated): Used to create a shipment for a return.
