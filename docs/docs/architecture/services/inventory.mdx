# Inventory Service

The inventory service consists of the context `ProductItem`.
It is responsible for managing the inventory - consisting of `ProductItems` of a `ProductVariant`. It is part of the order and return saga, where it reserves and releases the required `ProductItems`.

## Technology Stack

- **Language**: TypeScript
- **Framework**: NestJS
- **GraphQL Library**: @nestjs/graphql with @nestjs/apollo (code first)
- **Database**: MongoDB
- **Entity Management**: Mongoose

## Events

This section lists events that are defined or used by the inventory service.

### Published Events

<details>
<summary>
#### inventory/product/created
</summary>

This event is published when a new product is created.

```tsx
{
  id: string,
  internalName: string,
  isPubliclyVisible: boolean,
  defaultVariantId: string,
  categoryIds: string[]
}
```
</details>

<details>
<summary>
#### inventory/product-item/reservation-succeeded
</summary>

This event is published when all `ProductItems` for the `ProductVariants` of an `Orders` `OrderItems` were reserved successfully.

```tsx
{
  order: Order;
}
```
</details>

<details>
<summary>
#### inventory/product-item/reservation-failed
</summary>

This event is published when at least one `ProductItem` for the `ProductVariants` of an `Orders` `OrderItems` could not be reserved.

```tsx
{
  order: Order,
  failedProductVariantIds: string[]
}
```
</details>

### Subscribed Events

- [catalog/product-variant/created](/docs/docs/architecture/services/catalog#catalogproduct-variantcreated): Used to import `ProductVariant` from the `Catalog` bounded context.
- [order/order/created](/docs/docs/architecture/services/order#orderordercreated): Used to reserve the `ProductItems` for an `Order`.
- [payment/payment/payment-enabled](/docs/docs/architecture/services/payment#paymentpayment-enabled): Send Inventory to Fullfillment
- [payment/payment/payment-failed](/docs/docs/architecture/services/payment#paymentpayment-failed): Releases Product Items related to the order
- [shipment/shipment/status-updated](/docs/docs/architecture/services/shipment#shipmentshipmentstatus-updated): Updates the inventory status based on the shipment status
- [shipment/shipment/created](/docs/docs/architecture/services/shipment#shipmentshipmentcreated): Updates the inventory status based on the shipment status
- [discount/order/validation-failed](/docs/docs/architecture/services/shipment#discountordervalidation-failed): Releases Product Items related to the order
