# Payment Service

The payment service consists of the bounded contexts `Payment`, `PaymentInformation` and `PaymentMethod`.
It is responsible for handling the payment process and managing `Users` payment information. It is part of the order and return saga, where it captures the payments for `Orders` or refunds the payment after an `Return`.
It _is not a payment provider_ but simulates communicating with one via the `Simulation` Service.

### REST API

The service provides a REST API which is used by the external payment provider to inform about payment status changes.

## Technology Stack

- **Language**: TypeScript
- **Framework**: NestJS
- **GraphQL Library**: @nestjs/graphql with @nestjs/apollo (code first)
- **Database**: MongoDB
- **Entity Management**: Mongoose

## Events

This section lists events that are defined or used by the payment service.

### Published Events

<details>
<summary>
#### payment/payment/payment-enabled
</summary>

This event is published when all steps necessary to be able to process a payment were successfully completed.
It indicates, that other services can continue the order saga.

```tsx
{
  order: Order;
}
```
</details>

<details>
<summary>
#### payment/payment/payment-processed
</summary>

This event is published when the full payment amount was successfully captured.

```tsx
{
  order: Order;
}
```
</details>

<details>
<summary>
#### payment/payment/payment-failed
</summary>

This event is published when the payment fails finally after its retries.

```tsx
{
  order: Order,
}
```
</details>

### Subscribed Events

- [user/user/created](/docs/docs/services/user#userusercreated): Used to create default payment Methods for each user when created.
- [discount/order/validation-succeeded](/docs/docs/services/discount#discountordervalidation-succeeded): Used to start the payment process after the previous steps finished successfully.
