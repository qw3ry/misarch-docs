Welcome to the MisArch requirements and architecture documentation. It is based on the representable microservice reference architecture [MiSArch](https://misarch.github.io), but deviates from it in some aspects. It is tailored to research the relation between requirements and architecture. For the original reference architecture, please consult the original project.

### Documentation

See

- [Requirements](/docs/category/requirements) for the requirements of the system. They are divided into [Domain Requirements](/docs/docs/requirements/domain-requirements), [Technical Requirements](/docs/docs/requirements/technical-requirements), [Use Cases](/docs/docs/requirements/use-cases), and [Business Processes](/docs/docs/requirements/business-processes).
- [Architecture](/docs/category/architecture) for the architectural design of the system, including the [domain model](/docs/docs/architecture/domain-model), [component diagram](/docs/docs/architecture/component-diagram), as well as:
    - [Components & Services](/docs/category/components--services) for detailed documentation of every domain service or system components
    - [ADRs](/docs/category/adrs) for the architectural decisions
