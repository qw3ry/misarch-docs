# Catalog Service

The catalog service consists of the bounded contexts `Catalog`, `Category`, and `CategoryValue`.
It is responsible for managing products, their variants and versions, categories, and their respective characteristics and values.

## Technology Stack

- **Language**: Kotlin
- **Framework**: Spring Boot
- **GraphQL Library**: GraphQL Kotlin
- **Database**: PostgreSQL
- **Schema Management**: Flyway

## Events

This section lists events that are defined or used by the catalog service.

### Published Events

<details>
<summary>
#### catalog/product/created
</summary>

This event is published when a new product is created.

```tsx
{
  id: string,
  internalName: string,
  isPubliclyVisible: boolean,
  defaultVariantId: string,
  categoryIds: string[]
}
```
</details>

<details>
<summary>
#### catalog/product-variant/created
</summary>

This event is published when a new product variant is created.
This includes the default variant created when a product is created.

```tsx
{
  id: string,
  productId: string,
  currentVersionId: string,
  isPubliclyVisible: boolean
}
```
</details>

<details>
<summary>
#### catalog/product-variant-version/created
</summary>

This event is published when a new product variant version is created.
This includes the initial version created when a product variant is created.

```tsx
{
  id: string,
  name: string,
  description: string,
  version: number,
  retailPrice: number,
  createdAt: string,
  canBeReturnedForDays?: number,
  productVariantId: string,
  taxRateId: string,
  weight: number,
  mediaIds: string[]
}
```
</details>

<details>
<summary>
#### catalog/category/created
</summary>

This event is published when a new category is created.

```tsx
{
  id: string,
  name: string,
  description: string
}
```
</details>

<details>
<summary>
#### catalog/product/updated
</summary>

This event is published when a product is updated.

```tsx
{
  id: string,
  internalName: string,
  isPubliclyVisible: boolean,
  defaultVariantId: string,
  categoryIds: string[]
}
```
</details>

<details>
<summary>
#### catalog/product-variant/updated
</summary>

This event is published when a product variant is updated.

```tsx
{
  id: string,
  isPubliclyVisible: boolean
}
```
</details>

### Subscribed Events

- [tax/tax-rate/created](/docs/docs/services/tax#taxtax-rate-versioncreated): Used to import `TaxRate` from the `Tax` bounded context.
- [media/media/created](/docs/docs/services/media#mediamediacreated): Used to import `Media` from the `Media` bounded context.
