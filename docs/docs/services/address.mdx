# Address Service

The address service provides the bounded context `Address`.
It is responsible for managing addresses, both for users and of the vendor itself.

## Technology Stack

- **Language**: Kotlin
- **Framework**: Spring Boot
- **GraphQL Library**: GraphQL Kotlin
- **Database**: PostgreSQL
- **Schema Management**: Flyway

## Events

This section lists events that are defined or used by the address service.

### Published Events

<details>
<summary>
#### address/user-address/created
</summary>

This event is published when a new user address is created.

```tsx
{
  id: string,
  name?: Name,
  street1: string,
  street2: string,
  city: string,
  postalCode: string,
  country: string,
  companyName?: string,
  userId: string
}
```
</details>

<details>
<summary>
#### address/vendor-address/created
</summary>

This event is published when a new vendor address is created.

```tsx
{
    id: string,
    name?: Name,
    street1: string,
    street2: string,
    city: string,
    postalCode: string,
    country: string,
    companyName?: string
}
```
</details>

<details>
<summary>
#### address/user-address/archived
</summary>

This event is published when a user address is archived.

```tsx
{
    id: string,
    userId: string
}
```
</details>

### Subscribed Events

- [user/user/created](/docs/docs/services/user#userusercreated): Used to validate that addresses belong to an existing users
